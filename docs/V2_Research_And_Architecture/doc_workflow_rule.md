# AI Agent 强制行为规范

**核心法则: 代码与文档必须同步更新**

任何代表本项目的 Agent 在进行代码/功能开发、逻辑重构或系统调试时，必须严格执行以下三步原则：

1.  **事前说明**: 在每次修改前，先口头或在 `PROGRESS.md` 中说明意图。
2.  **事中实施**: 执行开发任务，创建/修改代码。
3.  **事后强刷(Mandatory Update)**: 开发任务结束后，**必须**同步更新项目的描述文件和进度报告：
    *   更新 `README.md` 中的系统架构或新增功能模块。
    *   **强制更新** `PROGRESS.md` 中的进度条 (Phase 状态)、任务打勾项和最后更新时间。
    *   将有价值的关键决策和技术方案补充进 `docs/` 相关文档，并同步写入长线记忆文件 (`MEMORY.md`)。

如果发现“进度不更新，只在背后默默写代码”的静默现象，本规则启动问责机制，视当天任务未完结。
